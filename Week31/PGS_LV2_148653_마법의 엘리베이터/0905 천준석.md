<접근법>
``` 
0. 해당 자리수가 5를 초과하면 위층(+)으로 올라가서 가는게 빠르고, 5미만이라면 아래층(-)으로 내려가서 가는게 빠르다
1. 만약 5라면? 해당 자리수 앞의 자리수가 5이상이라면 올라갔다 가는게 빠르기 때문에,올라가는 로직을 구성한다.
2. 내려가는 로직은 나머지를 더해주면 된다
ex. 조건2 예제 : 2555
1) 2555 -> 255 (5 : 내려감) -> 25 (5+5) -> 2 (5+5+5) -> 0 (5+5+5+2) => 17 (계속 내려가는 경우)
2) 2555 -> 256 (10-5 : 올라감) -> 26 (5+(10-6) : 올라감) -> 3 (5+4+(10-6) : 올라감) -> 0 (5+4+4+3 : 내려감) => 16(왔다갔다 -> 최적)
3. 해당 경우로 조건2가 필요하다.
```


```python
def solution(storey):
    answer = 0
    
    # 위치가 0이하로 내려갈때까지 반복문 처리
    while storey > 0:
        # 제일 뒷 자리수부터 확인하기 위해 10으로 나눠서, 현재 값 갱신 / 나머지로 나눈다.
        storey, rest = storey//10, storey%10
        # 위로 가는 조건
        # 1. 나머지(=마지막 자리수)가 5 초과라면 위로가는게 더 빠르다
        # 2. 나머지가 5이고, 바로 앞 자리수가 5이상이라면 위로가는게 더 빠르다
        # 조건2에서 왜 5초과가 아니고 이상일까? -> 만약 조건이 만족되면 storey 값이 1더하고, 5라면 6으로 변하기 때문에 5초과 위로 가는 조건 충족
        if rest > 5 or (rest == 5 and storey % 10 >= 5):
            # 위로 간다면, 역순으로 10을 빼주면 올라가는 층수가 나온다
            rest = 10 - rest
            # 10을 빼주었기 때문에, 바로 앞자리수에 1을 더해준다
            storey += 1
        # 결국 마지막 자리수가 층을 나타내니 결과값에 더해준다.
        answer += rest
    return answer
```
